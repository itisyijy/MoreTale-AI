# Adjunct Modules (조미료 프롬프트)

이 폴더는 핵심 프롬프트(`system_instruction.txt`, `style_guide.txt`, `user_prompt.txt`)에 사용자별·작품별로 선택적으로 주입되는 보조 프롬프트(adjunct modules)를 모은다. 핵심 프롬프트는 범용성을 유지하고, 디테일은 모듈로 분리한다.

## 구조

```
modules/
  language/   언어별 (한국어/영어/중국어/일본어/스페인어/베트남어)
  age/        연령대별 (3-5, 6-8, 9-12)
  culture/    문화권별 (각 국가/지역의 가족·음식·축제·가치관)
  gender/     성별 관련 가이드 (예: 고정관념 회피)
  family/     가정 상황 (한부모, 다세대, 입양 등)
  interest/   관심사 (동물, 우주, 공룡 등)
```

## 동작 방식

1. 백엔드가 사용자 프로필(아이의 나이·성별·가정·관심사·언어쌍·문화권)을 읽는다.
2. 해당하는 모듈 파일들을 골라 그 내용을 `user_prompt.txt`의 모듈 슬롯에 그대로 붙여 넣는다. 권장: 백엔드가 각 모듈 파일을 주입하기 전에 "## Apply when" 섹션은 strip한다(이 섹션은 백엔드용 메타 정보로, LLM 컨텍스트에 들어갈 필요가 없음).
3. gender 모듈은 특수: 항상 inclusive.txt가 함께 들어간다. 성별이 명시된 경우 [boys.txt 또는 girls.txt] + inclusive.txt 두 파일을 concat하여 단일 module_gender 슬롯에 주입. 성별 미명시 시 inclusive.txt 단독 주입.
4. LLM은 핵심 프롬프트의 보편 규칙 + 주입된 모듈의 컨텍스트별 규칙을 동시에 따른다. 충돌 시 시스템 프롬프트의 precedence 규칙이 적용된다.

## 모듈 작성 규칙

- 한 모듈 = 하나의 카테고리 한 가지 값. 파일명은 의미 그대로(예: `language/korean.txt`).
- 핵심 프롬프트와 충돌하지 않는 범위에서 **추가** 디테일만 담는다.
- "이 언어/연령/문화에서만 의미 있는 것"만 담는다. 보편적 조언은 핵심 프롬프트에 둔다.
- 형식은 자유 텍스트 또는 짧은 항목 나열. LLM이 읽고 따를 수 있으면 된다.
- 분량은 한 모듈당 30–80줄 권장. 너무 길면 LLM이 우선순위를 잃는다.

## 추가/수정

운영자는 핵심 프롬프트를 건드리지 않고 이 폴더에서만 모듈을 추가·수정할 수 있다. 새 카테고리(예: `season/`, `event/`)도 자유롭게 추가 가능하며, `user_prompt.txt`에 해당 슬롯만 추가하면 된다.
