# Backend Integration Guide

이 문서는 `MoreTale-AI` FastAPI 서버와 Spring Boot backend의 연동 기준을 정리한다.

## Swagger Source

- Swagger UI: `https://moretale-backend.onrender.com/swagger-ui/index.html#/`
- OpenAPI JSON: `https://moretale-backend.onrender.com/v3/api-docs`
- 로컬 스냅샷: `docs/moretale-backend-openapi.json`
- 확인일: 2026-05-17

OpenAPI spec은 Springdoc 기본 경로인 `/v3/api-docs`에서 직접 접근 가능하다. Swagger UI initializer는 `/v3/api-docs/swagger-config`를 사용한다.

## Auth Boundary

Backend public API는 Google OAuth2 이후 발급된 JWT를 사용한다.

```http
Authorization: Bearer {jwt}
```

AI 서버의 현재 내부 API는 backend-to-AI 호출용으로 `X-API-Key`를 사용한다.

```http
X-API-Key: {MORETALE_API_KEY}
```

즉 프론트엔드는 backend JWT만 다루고, AI 서버 API key는 backend 서버 환경변수에만 둔다.

## Story Generation Contract

Swagger의 `POST /api/stories/generate`는 동기 응답을 명시한다.

Request: `StoryGenerateRequest`

```json
{
  "prompt": "토끼와 거북이가 우주로 여행을 떠나는 이야기",
  "profileId": 1,
  "childName": "민준",
  "primaryLanguage": "ko",
  "secondaryLanguage": "vi",
  "ageGroup": "AGE_5_6",
  "childAge": 5,
  "firstLanguageProficiency": "BEE",
  "secondLanguageProficiency": "LARVA",
  "firstLanguageListening": "BEE",
  "firstLanguageSpeaking": "PUPA",
  "secondLanguageListening": "LARVA",
  "secondLanguageSpeaking": "EGG",
  "storyPreference": "FUN_ADVENTURE",
  "autoGenerated": false,
  "recommendedTaleTitle": "흥부와 놀부"
}
```

Response envelope: `ApiResponseStoryGenerateResponse`

```json
{
  "success": true,
  "data": {
    "title": "흥부와 놀부",
    "childName": "민준",
    "primaryLanguage": "ko",
    "secondaryLanguage": "vi",
    "slides": [
      {
        "order": 0,
        "imageUrl": "/static/outputs/{aiJobId}/illustrations/page_01.png",
        "textKr": "...",
        "textNative": "...",
        "audioUrlKr": "/static/outputs/{aiJobId}/audio/01_korean/page_01_primary.wav",
        "audioUrlNative": "/static/outputs/{aiJobId}/audio/02_vietnamese/page_01_secondary.wav"
      }
    ]
  },
  "message": null,
  "errorCode": null,
  "timestamp": "2026-05-17T00:00:00Z"
}
```

AI 서버의 `app.schemas.story.StoryGenerateRequest`와 `StoryGenerateResponse`는 이 camelCase 계약을 기준으로 맞춘다. `primaryLanguage`, `secondaryLanguage`는 backend 예시에 맞춰 `ko`, `vi`, `en` 같은 lowercase ISO 코드로 정규화한다. 추가로 `GET /api/stories/init` 응답에 있는 `familyStructure`, `customFamilyStructure`, `childNationality`, `parentCountry`는 AI 프롬프트 보강용 optional 필드로 받을 수 있다.

## TTS Contract

Swagger의 `TTSRequest`는 `text`, `language`, `style`을 받는다. AI internal TTS job도 같은 필드를 수용한다.

```json
{
  "text": "옛날 옛날에 흥부와 놀부가 살았어요.",
  "language": "ko-KR",
  "style": "neutral"
}
```

`language`는 backend 예시와 같은 canonical locale casing으로 정규화한다.

| Input | Output |
|---|---|
| `ko-kr`, `KO-kr` | `ko-KR` |
| `en-us` | `en-US` |
| `ja-jp` | `ja-JP` |
| `zh-cn` | `zh-CN` |
| `es-es` | `es-ES` |
| `vi-vn` | `vi-VN` |

응답의 `duration`은 저장된 WAV 파일을 기준으로 계산한 초 단위 정수다. 길이를 읽을 수 없는 파일은 `0`으로 반환한다.

## AI Server Routes

현재 AI 서버에는 두 계열의 연동 API가 있다.

| Route | Purpose |
|---|---|
| `POST /api/stories/` | AI 자체 story job 생성. `202` + status/result URL |
| `POST /api/stories/generate` | backend `StoryGenerateRequest`를 받아 AI job으로 변환. 현재 `202` |
| `POST /internal/ai/story/jobs` | callback URL이 포함된 backend-to-AI 비동기 story job |
| `POST /internal/ai/tts/jobs` | 단일/배치 TTS job |
| `POST /internal/ai/quiz/jobs` | quiz generation job |
| `POST /internal/ai/vocab/jobs` | vocabulary extraction job |
| `GET /internal/ai/jobs/{jobId}` | internal job status |
| `GET /internal/ai/{type}/jobs/{jobId}/result` | internal job result |

## Story Generation Direction

백엔드와 AI 서버의 story generation 연동은 **비동기 internal job**을 기준으로 한다.
콘텐츠 생성은 스토리 텍스트만 생성해도 오래 걸릴 수 있고, 이미지/TTS까지 포함하면 수 분 이상 걸리므로
백엔드가 AI에 동기 응답을 기대하지 않는다.

Backend side:

- 사용자가 생성 요청을 하면 backend 자체 job을 만든다.
- backend는 `POST /internal/ai/story/jobs`로 AI job을 생성한다.
- AI webhook 또는 backend polling으로 완료 상태를 반영한다.
- 완료 후 `GET /internal/ai/story/jobs/{jobId}/result`에서 `StoryGenerateResponse` 형태의 결과를 가져온다.
- story job 성공 결과에는 텍스트, 페이지 이미지 URL, 양쪽 언어 TTS URL이 함께 포함된다.

AI side:

- `POST /internal/ai/story/jobs`: `202 Accepted` + `{ jobId, statusUrl, resultUrl, callbackUrl }`
- `GET /internal/ai/jobs/{jobId}`: job status
- `GET /internal/ai/story/jobs/{jobId}/result`: completed/failed result
- `callbackUrl`로 완료/실패 webhook 전송
- 내부 story job은 TTS와 illustration을 항상 함께 생성한다.
- TTS 또는 illustration 중 하나라도 실패하면 partial success로 완료하지 않고 전체 AI job을 failed 처리한다. Backend는 같은 요청을 새 job으로 재시도한다.

기존 `POST /api/stories/generate`는 하위 호환용 public-style async endpoint로 유지하지만,
backend-to-AI 연동의 기준 endpoint는 `/internal/ai/story/jobs`다.

## Recommended Flow

비동기 backend job 흐름:

1. Frontend가 `POST /api/stories/generate`를 backend에 호출한다.
2. Backend가 backend-side story generation job을 만들고 frontend에 job id/status API를 반환한다.
3. Backend가 `POST /internal/ai/story/jobs`로 AI job을 만든다.
4. AI가 생성 완료/실패 시 `callbackUrl`로 webhook을 보낸다.
5. Backend가 `GET /internal/ai/story/jobs/{jobId}/result`로 결과를 가져와 DB에 저장/반영한다.
6. Frontend는 backend job status를 polling하거나, 추후 SSE/WebSocket으로 상태를 받는다.

## Page Count and Slide Order

Story page count는 backend 입력값으로 받지 않는다. AI가 reader profile을 기준으로 내부 정책에 따라 정한다.

현재 backend `ageGroup` 기준:

| ageGroup | pageCount |
|---|---:|
| `AGE_0_2` | 8 |
| `AGE_3_4` | 12 |
| `AGE_5_6` | 16 |
| `AGE_7_8` | 24 |
| `AGE_9_10` | 32 |
| `AGE_10_PLUS` | 32 |

`ageGroup`이 없으면 `childAge`로 같은 bucket을 계산하고, 둘 다 없으면 age 5 기준인 16 pages를 사용한다.

AI 내부 `Story.pages[].page_number`는 generation/model layer에서 1-based를 유지한다. Backend contract로 반환하는 `slides[].order`는 DB 저장 기준에 맞춰 **0-based**로 변환한다.
